2.border-radius

border-radius: 20px  =  border-top-left-radius: 20px    =  border-top-left-radius: 20px 20px    注：分别是x轴和y轴
                        border-top-right-radius: 20px      border-top-right-radius: 20px 20px
                        border-bottom-right-radius: 20px   border-bottom-right-radius: 20px 20px
                        border-bottom-left-radius: 20px    border-bottom-left-radius: 20px 20px

3.box-shadow 和 text-shadow

box-shadow: X轴偏移量 Y轴偏移量 [阴影模糊半径] [阴影扩展半径] [阴影颜色] [投影方式];
例. box-shadow: 4px 2px 6px 7px #333333 inset;

text-shadow: X-Offset Y-Offset blur color
例. text-shadow: 0 1px 1px #fff
X-Offset：表示阴影的水平偏移距离，其值为正值时阴影向右偏移，反之向左偏移；
Y-Offset：是指阴影的垂直偏移距离，如果其值是正值时，阴影向下偏移，反之向上偏移；

4.rgba

color：rgba(R,G,B,A)
R、G、B三个参数，正整数值的取值范围为：0 - 255,百分数值的取值范围为：0.0% - 100.0% ;
A为透明度参数，取值在0~1之间，不可为负值。

5.css3渐变

(1)线性渐变 linear-gradient([direction], color[percent], color[percent], …);

direction为选填，角度的单位为 deg ，也可以用to bottom, to left, to top left等的方式来表达

(2)径向渐变 radial-gradient(shape at position, color[percent], color[percent])

shape:放射的形状，可以为原型circle，可以为椭圆ellipse;
position: 圆心位置，可以两个值，也可以一个，如果为一个时，第二个值默认center 即 50%。值类型可以为，百分数，距离像素，也可以是方位值(left,top...);

8.border-image

border-image: source slice width outset repeat;

    source : 用于指定要用于绘制边框的图像的位置
    slice  : 图像边界向内偏移
    width  : 图像边界的宽度
    outset : 用于指定在边框外部绘制 border-image-area 的量
    repeat : 用于设置图像边界是否应重复（repeat）、拉伸（stretch）或铺满（round）

9.background

(1)background-origin
可选的值：
padding-box(默认) :背景图像相对于内边距框来定位
border-box :背景图像相对于边框盒来定位
content-box :背景图像相对于内容框来定位 

它只相当于规定图片的左上角以什么地方为原点，其他的它就不负责了

(2)background-clip
可选的值：
border-box(默认) :背景被裁剪到边框盒
padding-box :背景被裁剪到内边距框
content-box :背景被裁剪到内容框

(3)background-size
可选的值：
length 设置背景图像的高度和宽度,第一个值为宽度，第二个值为高度,如果只设一个值，则第二个值会被设置为”auto”
percentage 以父元素的百分比来设置背景图像的宽度和高度
cover 用一张图片铺满整个背景，如果比例不符，则截断图片
contain 尽量让背景内，存在一整张图片

10.属性选择器

E[att^=”val”]{…} 选择匹配元素E, 且E元素定义了属性att, 其属性值以val开头的任何字符串

E[att$=”val”]{…} 选择匹配元素E, 且E元素定义了属性att, 其属性值以val结尾的任何字符串

E[att*=”val”]{…} 选择匹配元素E, 且E元素定义了属性att, 其属性值任意位置出现了val,位置不限

例. div[class*="test"]{
        background:#ffff00;
    }

11.伪类选择器

(1):root{} 跟标签选择器，等同于html{}

(2):not([att=”val”]){} 否定选择器,排除某些特定条件的元素

(3):empty{} 空标签选择器,用来选择没有内容的元素、不在文档树中的元素

(4):target{} 目标元素选择器,用来匹配被location.hash 选中的元素(即锚点元素)

(5):first-child{} 第一个子元素
:last-child{} 最后一个子元素
:nth-child(n){} 第n个子元素
:nth-last-child(n){} 从后往前数

(6):first-of-type{} 第一个该类型的子元素
:last-of-type{} 最后一该类型的个子元素
:nth-of-type(n){} 第n个该类型的子元素
:nth-last-of-type(n){} 从后往前数

:first-child 和 :first-of-type的区别：
  :first-child 匹配的是父元素的第一个子元素，结构上的
  :first-of-type 匹配的是该类型的第一个元素

(7):only-child 和 :only-of-type

前者选择的是独生子的子元素，结构上的；后者是选择该类型的独生子子元素

(8):enabled 和 :disabled 、:checked
在web的表单中，有些表单元素有可用(“enabled”)和不可用(“disabled”)状态，比如输入框，密码框，复选框等。

:checked{} 选择框的被选中状态下的元素

(9):read-only{} 和 :read-write{}

前者选中只读的元素，后者选中非只读的元素
比如< input type="text" readonly="readonly"/> 为只读元素

12.伪元素选择器

css3增添了一个 ::selection，用来匹配突出显示的文本（用鼠标选中的文本）。默认情况下，选中文本是以蓝色的背景，白色的字体显示。

13.条件选择

div>span  div标签的下一层中的span标签
div+span  div标签后面紧跟的兄弟span标签
div~span  div标签后面的所有兄弟span标签

14.transform

transform: none | tranform-functions

tranform-functions如下：

rotate(angle)  以x/y/z为轴进行旋转，默认为z，rotatex(), rotatey(), rotatez(), rotate3d(x, y, z, angle)

scale(x, y)   接受两个值，如果没有第二参数，则第二个参数使用第一个参数的值，值是数字表示倍数，不加任何单位
              scalex(),scaley() scalez() scale3d(sx,sy,sz)

skew(x, y)    定义沿着 X 和 Y 轴的 2D 倾斜转换。如果第二个参数未提供，则默认值为0 。skewx(), skewy()

translate(x, [y])   可以移动距离,相对于自身位置
                    translatex(),translatey(),translatez(),translate3d(x, y, z)

transform-origin: x-axis y-axis z-axis;  变换原点，默认50% 50% 0 即盒子的中心，该属性要和transform一起使用。

15.transition动画

transition: property duration timing-function delay;

transition-property    规定设置过渡效果的 CSS 属性的名称。
transition-duration    规定完成过渡效果需要多少秒或毫秒。
transition-timing-function    规定速度效果的速度曲线，例如linear(默认)、ease
transition-delay    定义过渡效果何时开始。

16.animation 动画铺垫

animation: name duration timing-function delay iteration-count direction;

animation-name          规定需要绑定到选择器的 keyframe 名称
animation-duration      规定完成动画所花费的时间，以秒或毫秒计
animation-timing-function    规定动画的速度曲线，例如linear(默认)、ease
animation-delay          规定在动画开始之前的延迟
animation-iteration-count    规定动画应该播放的次数，infinite/number
animation-direction      规定是否应该轮流反向播放动画

动画关键帧 @keyframes 0% - 100% 代表动画的时间过渡
    例如@keyframes demoMove{
            0%{ background-color:red;}
            10%{ background-color:green;}
            20%{ background-color:white;}
            50%{ width:200px;}
            100%{ height:200px;}
        }
帧频里面如果只有 0% 和 100%两个关键帧，那么可以用 from to 代替

17.columns 多列布局

columns: [column-width] [column-count];
column-width 指每一列的宽度 根据容器宽度自适应（最小宽度）
column-count 指规定的列数 唯一精准的是列数
不要两一起使用 会乱

column-gap 设置列与列之间的宽度(间隙)，直接用数值表示，如不设置则默认按 font-size 来设置

column-rule: width style color  设置列与列之间间隙的分界线样式，style有none、hidden、dotted、
                                dashed、solid、double、groove、ridge、inset、outset

column-span：1/all  用于规定标题可以跨越几列，all就是全都跨过

18.盒模型

(1)W3C标准盒模型下width不包括padding 和 border，而IE6混杂模式盒模型下，width包含padding 和border。

box-sizing : border-box/content-box

    content-box为W3C标准盒子
    border-box为IE6混杂模式的盒子

(2)弹性盒子

在父元素上(容器上)设置 display: flex; flex为复核属性

    flex-direction 决定主轴的方向（即子元素的排列方向），可能的值 row(水平显示) row-reverse(水平反向)
                                                              column(垂直显示) column-reverse

    flex-wrap  定义如果一条轴线排不下，如何换行。可能的值 nowrap(不换行) wrap(换行) wrap-reverse(反向换行)

    flex-flow   flex-direction 和 flex-wrap 的简写形式，默认值为row nowrap

    justify-content  定义了子元素在主轴上的对齐方式，可能的值 flex-start(默认，容器开头) flex-end(容器结尾)
                                                          center(容器中心) space-between(元素间留空白) 
                                                          space-around(元素中间和两侧都有空白)

    align-items  定义子元素在交叉轴上如何对齐，可能的值 flex-start(交叉轴的起点对齐) flex-end(交叉轴的终点对齐)
                                                    center(交叉轴的中点对齐) baseline(第一行文字的基线对齐)
                                                    stretch(默认,若子元素未设置高度或设为auto，占满整个容器)

    align-content  定义了多根轴线的对齐方式，值与align-items相同


在子元素上设置 flex: flex-grow flex-shrink flex-basis;  默认值为0 1 auto，后两个属性可选

    flex-grow 默认为0，当有多余空间时，该元素能分配到几份

    flex-shrink 默认为1，如果所有项目的flex-shrink属性都为1，当空间不足时，都将等比例缩小。
                如果一个项目的flex-shrink属性为0，其他项目都为1，则空间不足时，前者不缩小。

    flex-basis  定义了在分配多余空间之前，该元素占据的主轴空间。它的默认值为auto，即项目的本来大小。
                它可以设为数值，如350px，则该子元素将占据固定空间。

    还有 order 定义项目的排列顺序。数值越小，排列越靠前，默认为0

        align-self 允许该子元素有与其他子元素不一样的对齐方式，可覆盖align-items属性，默认值为auto，
               表示继承父元素的align-items属性，如果没有父元素，则等同于stretch。

20.transform-style 3d变换

定义被嵌套元素如何在 3D 空间中显示，需要设置在父元素中。
    注：设置了transform-style:preserve-3d的元素，不能设置overflow：hidden；否则preserve-3d失效。

可选的值：flat   子元素将不保留其 3D 位置
         preserve-3d  子元素将保留其 3D 位置

景深 perspective：默认none  设置在父元素上，也就是父元素的景深
     perspective-origin:  默认值是50% 50%

    perspective：60px 与  transform: perspective(600px) 的区别：
    前者用在舞台元素上(变形元素们的共同父元素)，后者就是用在当前变形元素上，并且可以与其他的transform函数一起使用。

backface-visibility: visible | hidden  定义是否展示元素的背面。

21.媒体查询

媒体查询是向不同设备提供不同样式的一种方式，它为每种类型的用户提供了最佳的体验。

(1)媒体查询的引用方法 link标签 @import @media
< link rel=”stylesheet” media=”screen and (max-width: 600px)” href=”1.css”/>
注：screen前加 not表示排除，加only用于向早期浏览器提供隐藏媒体查询

@import url(1.css) screen and (max-width:360px)

@media all and (max-width: 1000px) and (min-width: 600px){}

(2)device-width/height 定义输出设备的屏幕可见宽/高度 (不是浏览器的宽/高度)
width/height使用documentElement.clientWidth/Height即viewport的值。

(3)物理像素 和 设备独立像素

一个物理像素是显示器(手机屏)上最小的物理显示单元，在操作系统的调度下，每一个设备像素都有自己的颜色值和亮度值。

设备独立像素(也叫目睹无关像素)，可以认为是计算机坐标系统中得一个点，这个点代表一个可以由程序使用的虚拟
像素（css像素），然后由相关系统转换为物理像素。

设备像素比(dpr) = 物理像素 / 设备独立像素

在不同的屏幕上（普通屏幕 vs retina屏幕），css像素所呈现的大小（物理尺寸）是一致的，不同的是一个css像素所对应
的物理像素个数是不一致。

22.浏览器渲染原理

(1)过程

1.处理HTML生成DOM树
浏览器将获得的HTML代码解析生成一个DOM树，html中的每个标签都将成为DOM树中的一个节点，根节点是document对象，树中也包括js中动态生成的DOM节点；

2.处理CSS生成CSSDOM树
浏览器把所有样式解析成结构体，在解析过程中去除掉不能识别的样式，生成CSSDOM树；

3.将两树合并成render树
DOM树和CSSDOM树合并成render树，render树中每个node都有自己的样式style，而树中不包括隐藏的节点(如display: none的节点，无样式的head节点)，这些节点不影响布局layout，但visibility: hidden 的节点会包含在render树上，因为其会影响布局，占用空间；

4.对render树进行布局计算
render树按照样式计算布局，布局阶段输出的结果称为"盒模型"，盒模型精确表达了窗口中每个元素的位置和大小，而且所有相应的度量单位都会转化成屏幕上的绝对像素位置；

5.将render树上的每个节点绘制到屏幕上
将这些信息渲染在屏幕的每个真实像素点上，这个阶段称为"绘制"，亦称为"栅格化"。

(2)重排 和 重绘

重排：在计算各节点在当前设备上的准确位置和尺寸时，这些布局需要做的事，称为回流(重排)，当render树种部分或全部
      元素的尺寸、位置布局或者隐藏等发生改变时，就会回流。每个页面至少回流一次，就在页面第一次加载时。

重绘：render树中一些元素节点的属性发生改变，而这些属性只影响元素的外观、风格，而不影响布局，这时会发生重绘。

重排和重绘影响性能
以下属性和方法需要返回最新的布局信息，重新计算render树，造成回流，应尽量合并在一起操作，减少回流次数。
这些属性包括：offsetTop offsetLeft offsetWidth offsetHeight；scrollTop scrollLeft scrollWidth scrollHeight；                        clientTop clientLeft clientWidth clientHeight；getComputedStyle() currentStyle()

想要提高网页性能，就需要降低重排、重绘的频率和成本，尽量少触发重新渲染。DOM变动和样式变动都会触发重新渲染，但如今的
浏览器已经很智能了，会把所有的变动集中起来，拍成一个队列，一次过执行，尽量避免多次重新渲染。

如果在样式的写操作之后，发生了上述触发重排属性的读操作，就会引发浏览器立即重排，这些重排不会形成队列优化。
例.  反面教材：div.style.left = div.offsetLeft + 'px';
              div.style.top = div.offsetTop + 'px';
              //触发了两次重排重绘

    正面教材：var left = div.offsetLeft + 'px';
             var top = div.offsetTop + 'px';
             div.style.left = left;
             div.style.top = top;
             //只触发一次重排重绘

性能优化的方法：
(1)DOM的多个读操作/写操作，应该放在一起写，不要两个读操作之间加了一个写操作;
(2)离线操作DOM，如使用隐藏元素 document.creatDocumentFragment() cloneNode();
(3)修改样式的时候添加类名，或一次性添加到 dom.style.cssText上。
